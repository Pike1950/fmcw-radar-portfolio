1. Answering: Why GND and not GNDD with a Ferrite Bead?

This is a great question and the answer is central to mixed-signal PCB design.

The ferrite bead splits the supply rail (3V3A vs 3V3D) — not the ground. The ground plane stays unified as a single, unbroken copper pour on L2. This is the correct topology for SAR ADC designs, and it's what TI recommends in the ADS8881 datasheet and application notes.

Why split the supply but not the ground?

Digital switching noise on the supply rail is conducted — the MCU draws current pulses from 3V3 every clock edge, and those pulses create voltage ripple. The ferrite bead's 220Ω impedance at 100MHz blocks this conducted noise from reaching the analog supply. That's its entire job.

Ground is different. Ground noise is dominated by return currents flowing through the ground plane. If you split the ground into GNDA and GNDD, you force return currents to flow through whatever narrow bridge connects them, creating a shared impedance that actually couples more noise than a solid plane. A single, solid L2 ground plane gives every return current the lowest-impedance path back to its source, which minimizes ground bounce.

The rule: one ground, split supplies. The ferrite bead isolates 3V3A from 3V3D. Both supplies return to the same GND plane. The layout enforces separation — analog components sit over one region of the ground plane, digital components over another, and you control the return current paths by where you place the components and route the traces, not by cutting the ground.

2. Actual ERC Errors (4 errors from your report)

Error 1: [hier_label_mismatch] — Sheet B missing 3V3A hierarchical label

Root sheet at (78.74, 45.72): The Sheet B symbol (Baseband_IF_Input_Protection_Bias) has a 3V3A pin defined on it in the root schematic, but inside Sheet B itself, there is no matching 3V3A hierarchical label.

Sheet B currently has only two hierarchical labels: VCM (input) and IF_BIASED (output). The 3V3A pin on the root sheet symbol expects a matching hierarchical label inside Sheet B.

Fix: Sheet B doesn't actually use 3V3A directly (R_BIAS connects to VCM, not to a supply), so you have two options:

  1. Remove the 3V3A pin from Sheet B's symbol on the root schematic (right-click the sheet symbol → Properties → delete the 3V3A pin). This is the cleaner option if Sheet B truly doesn't need the supply rail.
  2. Add a matching 3V3A hierarchical label inside Sheet B. You'd only do this if Sheet B needs 3V3A for something (e.g., if you later add a bypass cap or pull-up on Sheet B).

Error 2: [power_pin_not_driven] — GND at ST1 area needs PWR_FLAG

Sheet A at (39.37, 33.02): A power:GND symbol exists here (near the Phoenix terminal ST1) but KiCad sees no power source driving the GND net. This is a common KiCad quirk — the screw terminal connector's pins are likely typed as "passive" in the symbol, so KiCad doesn't recognize that external power enters through them.

Fix: Place a power:PWR_FLAG symbol on the GND net, right at the ST1 ground pin. Also place one on the 5V input net at ST1 if you get a similar warning there. PWR_FLAG tells KiCad "yes, this net is intentionally powered from an external source." It doesn't affect the netlist or layout — it's purely an ERC annotation.

Error 3: [pin_not_connected] — BAT54S D1 Pin 1 (Anode)

Sheet A at (80.01, 30.48): The BAT54S Pin 1 is flagged as unconnected. The BAT54S in SOT-23 has three pins. Depending on the SamacSys symbol's pin mapping, Pin 1 might be the common anode or one of the cathodes.

Fix: Check the symbol's pin assignments against the BAT54S datasheet. In the standard SOT-23 package: Pin 1 = Common Anode, Pin 2 = Cathode 1 (for one diode), Pin 3 = Cathode 2 (for the other diode). For reverse polarity protection, the common anode typically connects to GND and one cathode connects in series with the input. If Pin 1 is the anode and it's truly floating, wire it appropriately. If the SamacSys symbol has a different pin mapping, verify which pin is actually unconnected and whether the protection circuit is complete.

Error 4: [pin_not_connected] — REF5025 U_REF1 Pin 5 (TRIM/NR)

Sheet A at (138.43, 73.66): The REF5025's TRIM/NR pin is unconnected. The datasheet says this pin can be left floating for default 2.500V output, or connected to an external capacitor for noise reduction.

Fix: Two options:

  1. Add a no-connect marker (X) on Pin 5 if you want the default 2.500V output with no external noise filter. This tells ERC the pin is intentionally unconnected.
  2. Add a noise reduction capacitor (the datasheet recommends 1–10 µF for the NR function) between TRIM/NR and GND. This filters the internal bandgap reference noise and can reduce output noise by ~60%. Given that this is a precision ADC application, you may want this — but verify the REF5025 datasheet's NR pin specifications for recommended cap values and settling time impact.

Warning 2: VOCM / VCM net name conflict on Sheet D

Sheet D: The hierarchical label VCM at (60.96, 76.2) and the net label VOCM at (68.58, 76.2) are on the same wire. KiCad warns that when two different names are on the same net, it picks one — in this case, VOCM wins and will be the net name in the netlist.

This is functionally correct — the VCM signal from Sheet A reaches the FDA's VOCM pin as intended. But the warning means that in your netlist and PCB layout, this net will appear as "VOCM" rather than "VCM". Any other sheets referencing VCM through the hierarchy will see a different net name than what appears locally on Sheet D.

Fix (optional): If you want a clean ERC, remove the VOCM net labels on Sheet D and just let the hierarchical label VCM name the net. The FDA's VOCM pin connects to whatever net the wire carries — the net label doesn't need to match the pin name. Alternatively, if you prefer the local clarity of seeing "VOCM" near the FDA, you can leave it and accept the warning. It won't cause any electrical problem.
