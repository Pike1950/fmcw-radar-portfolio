2. Critical Design Decision: PGA113 DVDD → 3V3A (Not 3V3D)

⚠ Both AVDD (Pin 1) and DVDD (Pin 10) connect to 3V3A — NOT 3V3D. Despite the "D" in DVDD suggesting digital, this pin powers the analog output stage that drives VOUT (Pin 5). Connecting it to the noisy post-ferrite 3V3D rail would inject MCU switching noise directly into the signal path.

What DVDD Actually Powers

1. Analog output stage — the final amplifier driving VOUT. This is your signal path.

2. SPI interface logic — registers for SCLK, DIO, ~CS. Draws only µA during transactions.

Why 3V3D Would Be Harmful

3V3D comes after ferrite bead FB1 (BLM18PG221SN1D, 220Ω at 100 MHz). This LC network isolates digital noise from analog. Feeding 3V3D into DVDD would couple MCU SPI clock edges, GPIO toggles, and digital activity directly into PGA113's output amplifier — completely bypassing the ferrite isolation designed on Sheet A.

When Would You Use Different AVDD/DVDD Voltages?

Only in mixed-voltage systems: AVDD=5V (max output swing), DVDD=3.3V (MCU logic). Not our case — entire system is 3.3V, so both pins get 3V3A.

What Is 3V3D Actually For?

Reserved for purely digital components: J_DFT header VCC, SPI pull-ups, debug LEDs, digital test infrastructure. These don't process analog signals, so supply noise is inconsequential.

DVDD Supply Choice
The ADS8881 has separate AVDD and DVDD pins. DVDD can run from 1.65V to 3.6V independently of AVDD. In this design, both are tied to 3V3A to keep a common ground return and avoid digital noise coupling. If the SPI bus is long or noisy, DVDD could alternatively connect to 3V3D — but then ensure the GND return paths don't create a ground loop through the ADC's shared GND pin.
DIN Pin — Interface Mode Selection
The ADS8881 supports three interface modes selected by the DIN pin state at the CONVST rising edge. In this design, DIN is connected to the SPI_SDI net, enabling 4-wire SPI mode. The MCU can dynamically select the mode by driving DIN high (CS mode) or low (daisy-chain mode) at the CONVST rising edge. For simple single-ADC operation, the MCU drives DIN high before asserting CONVST — equivalent to 3-wire CS mode but with the flexibility to switch to daisy-chain later without hardware changes.

8.2 Test Pin Bed — J_DFT Header + MMCX Probe Points

Design intent: Provide three probe access methods for every critical signal: (1) bare pads for scope tip probes and pigtail ground clip connections during quick checks, (2) MMCX connectors for hands-free scope connection with controlled impedance, and (3) a consolidated GPIO header (J_DFT) that brings all monitor points plus the external MCU's SPI signals to a single connector for automated test and firmware-driven monitoring.

8.2.1 J_DFT — Test/Monitor Header (2×10, 2.54mm pitch)

J_DFT is the single external connector for both SPI operation and test/monitoring. The 2.54mm pitch is chosen specifically to allow direct connection to commodity MCU eval boards (e.g., ESP32-S3-DevKitC-1) via dupont jumper wires or a pin-to-pin adapter. The external MCU drives the PGA113 and ADS8881 via the SPI pins on this header while simultaneously monitoring analog signals via its ADC inputs.

Analog signal isolation: Pins 7, 8, 15, 16, 17, 18 carry analog signals that must not be loaded by external MCU GPIO pins during normal operation. Each of these is connected through a 10kΩ series isolation resistor (0402) placed at the PCB near the source signal. The 10kΩ in combination with the MCU ADC's input capacitance (~5–15pF) forms a pole at ~1–3 MHz — well above the 15 kHz signal bandwidth, so it doesn't affect measurements. But it prevents any bus capacitance or MCU pin leakage from disturbing the main signal chain. Digital pins (9–13) connect directly with no series resistor.

8.2.3 REF5025 TEMP Pin Routing (U_REF1 Pin 3)

The REF5025 TEMP pin (Pin 3) outputs a voltage proportional to die temperature: approximately 1.85 mV/°C with ~557 mV at 25°C. This is useful for two purposes: (1) correlating VREF drift with temperature during thermal testing, and (2) providing a board-level temperature indication for the MCU's AGC firmware (gain calibration can be temperature-indexed).

8.3 Phased Bring-up Order

Phase 1: Power (Sheet A only) — Verify 3V3A, VREF, VCM, measure quiescent current (expect 2–4 mA). Read TEMP pin via J_DFT to confirm REF5025 is alive.
Phase 2: Input (Add Sheet B) — Verify DC bias at PGA input = 1.65V. Inject test signal, confirm AC coupling works.
Phase 3: PGA + Filter (Add Sheet C) — Sweep all 8 gain steps. Run filter Bode plot (100 Hz–200 kHz). Use J_MMCX_PGA1 and J_MMCX_LPF2.
Phase 4: FDA + ADC (Add Sheet D) — Verify FDA VCM, first ADC readout via CONVST on J_DFT, then 8192-sample FFT capture.
Phase 5: System — Full dynamic range, noise floor at G=200×, alias rejection at 55 kHz, total current 6–8 mA. Use J_DFT for automated test via external MCU.

8.6 Functional Test Controller — ESP32-S3-DevKitC-1 Integration

Concept: An ESP32-S3-DevKitC-1 (~$10, 70×28mm) connects to J_DFT via dupont jumper wires or a 20-pin ribbon jumper harness. The ESP32 runs a built-in self-test (BIST) firmware that exercises every block in the signal chain and reports pass/fail over USB serial (bench debug) or WiFi (fab-floor test station). J_DFT is the single external interface — the ESP32 drives SPI for PGA113/ADS8881 operation and monitors all analog points through the same connector.

8.6.3 Physical Integration

ItemDetail
Connection method20× individual dupont jumper wires from J_DFT pins to DevKitC header pins, or a 20-wire ribbon jumper harness with 2.54mm single-row housings on each end. For production, a custom wiring harness with a 2×10 IDC connector on the J_DFT side and split-out individual pins on the ESP32 side.
PowerThe ESP32-S3-DevKitC is powered independently via its own USB-C port — do not power it from J_DFT's 3V3A pin. The 3V3A and 3V3D connections on J_DFT are monitor-only (read via ADC). This keeps the baseband board's clean analog supply completely isolated from the ESP32's noisy digital supply.
GroundJ_DFT provides 4 GND pins (2, 4, 14, 20). Connect at least two to the ESP32's GND for a low-impedance return path. The shared ground ensures ADC readings have a common reference.
SPI bus ownershipJ_DFT is the sole SPI connection point. Only one external controller should be connected at a time. If a different MCU is used for production vs. test, swap the jumper harness between controllers — do not drive SPI from two sources simultaneously.
Test fixture conceptFor fab-level functional test: mount the baseband board in a simple acrylic jig with pogo-pin contacts on J_DFT pads (no header soldered). The ESP32 sits on a separate fixture plate with permanent wiring to the pogo pins. Operator inserts board, presses down, BIST runs in ~3 seconds, LED goes green/red. Throughput: ~15 boards/minute with one fixture.

10.4 Direction Conventions

SPI signal directions — Output on Sheet D ✅
The MCU is off-board (external), so it doesn't exist in the schematic hierarchy. Within the design, J_DFT on Sheet D is where SPI_SCLK, SPI_SDI, and CS_PGA originate — they enter through the connector and fan out from there. Sheet D is the source, Sheet C's PGA113 is the sink. All three signals now correctly use Output on Sheet D and Input on Sheet C.
Power nets: Passive vs Output/Input
Although the LDO on Sheet A physically generates 3V3A, KiCad treats power distribution as non-directional. Using Passive on all sheets avoids DRC warnings. GND uses global power symbols (power:GND) on all sheets — no hierarchical labels needed.
Multiple hier labels with the same name on one sheet
Placing 3V3A as a hierarchical label 2 times on Sheet D is correct — they all connect to the single 3V3A sheet pin on the root. Each instance is just a tap point for distributing the power rail to different physical locations within the sheet. Additional taps use local net labels (3 on Sheet D).
