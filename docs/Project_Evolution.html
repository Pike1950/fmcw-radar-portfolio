What Went Wrong

Assembly Failures

The ADS1174 (64-pin TQFP) and TPS717185 (sub-mm pin pitch) both had solder bridges during hand assembly. After desoldering and rework, components were moved to a single RX section on a fresh board — but power routing only connected through the RX_3 section, requiring wire jumpers to supply the RX_1 section being tested.

Initial testing showed signal passing through VGA → LPF → SE-to-Diff. The next day, the signal path failed — suspected IC damage from repeated rework or an undetectable short. COVID-19 shutdown (March 2020) prevented further debugging.

Root cause: No test infrastructure (zero test points, no DFT header, no isolation capability), no systematic bringup plan, and component packages too aggressive for hand assembly.

Issues Caught Before Fabrication

The systematic review process caught these errors that would have required board respins:

  1. C14 grounding error — Sallen-Key feedback cap connected to GND instead of op-amp output (would have killed filter response)
  2. PGA113 DVDD assignment — Would have placed digital supply noise on the analog output stage
  3. ADS8881 DIN pin mode — Wrong SPI mode (3-wire vs 4-wire); ADC would not communicate
  4. Duplicate VREF hierarchical label — Conflicting signal directions on Sheet D
  5. VCM voltage correction — Changed from 1.25V to 1.65V (VDD/2) for 2.7 dB SNR improvement
  6. Multiple dangling nets — Unconnected hierarchical labels, missing wires on MMCX connectors

3. Career Context — The Shield AI Interview

Applied for Senior Electrical Engineer, Hardware Test (R4000) at Shield AI, Dallas. Panel interview Feb 18, 2026. Presented the Rev A → Rev B redesign story.

Outcome

Not selected. Feedback from recruiter (Jim):

Strengths identified: Effort appreciated, solid awareness of common design practices, understanding of ground loop inductance reduction.
Gaps identified (verbatim feedback):

"Depth & immediacy needed in unpacking concepts like capacitor nets paired together, impedance vs frequency tradeoffs for passive components, high-speed board design without reference designs."

"Strengthening the 'Why'" — the main feedback was that first-principles reasoning needed to be immediate and confident. Specifically: signal integrity, relay driver protection, impedance-based component placement, layout reviews from a system perspective.

Assessment

The feedback accurately identifies a gap between IC-level validation experience (TI: validating other people's designs) and system-level PCB design fluency (Shield AI: designing test systems from scratch without reference designs). The gap is closable — it's an experience gap, not an ability gap. The portfolio and project work below are specifically designed to close it.

Phase 1B: Schematic Updates & Freeze

HARDWARE DOC Before layout begins, evaluate whether the first-principles documentation or the digital platform decision (FPGA/MCU selection) requires any schematic changes. Freeze the schematic only after all updates are verified.

Digital platform connector review — Once FPGA board is selected (Pico-Ice, Tang Nano 9K, or other), verify that J_DFT pinout and J_SPI header are compatible with the chosen platform's GPIO mapping. Update pin assignments if needed.
FPGA/MCU header evaluation — Determine whether a dedicated FPGA header (separate from J_DFT) should be added, or if J_DFT and J_SPI are sufficient for both debug and production use. Consider adding a PMOD-compatible footprint for direct FPGA board mating.
Documentation-driven corrections — As Sheets A–D are documented with first-principles analysis, any component value changes, additional decoupling, or topology adjustments identified during the deep-dive get incorporated into the schematic.
Final ERC pass + schematic freeze — Run ERC, verify 0 errors/0 warnings, update component count, tag the KiCad project as "Release Candidate" for layout. No schematic changes after this point without a formal change order.

Phase 2: PCB Layout

HARDWARE Route the board in KiCad. This is the single highest-value activity — it directly demonstrates the skills Shield AI was testing for.

Stackup definition — 4-layer: L1 Signal, L2 GND (continuous), L3 Power (3V3A/3V3D zones), L4 Signal (DFT).
Impedance calculations — Trace widths for 50Ω microstrip on the stackup. Saturn PCB Toolkit or KiCad calculator.
Component placement — Signal flows left→right, power top, digital/DFT bottom, FDA+ADC cluster ≤15mm.
Critical routing — FDA→ADC diff pair matched, SCLK damping R within 3mm, decoupling ≤2mm from IC pins.
Layout design review document — Written like a TI design review, explaining every placement and routing decision.

Phase 3: Relay Driver Protection Mini-Project

HARDWARE Self-contained project directly addressing Shield AI's "relay driver protection" feedback. MOSFET driving inductive load with flyback diode + snubber, simulated in LTspice.

Phase 4: SPI Controller in SystemVerilog

DIGITAL Design an SPI master that interfaces with the ADS8881 and PGA113. Connects the microprocessor architecture coursework to the radar system.

RTL design — SystemVerilog SPI master module with configurable CPOL/CPHA, ADS8881 acquisition timing.
Verification — Self-checking constrained-random testbench (extending ALU verification skills to real peripherals).
Synthesis — Target FPGA TBD (Pico-Ice iCE40UP5K at $35, or Tang Nano 9K at $20).
Hardware validation — Bench verification of SPI timing on actual board (TI validation skills applied to own design).
